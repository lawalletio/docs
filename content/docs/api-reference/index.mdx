## Status

The full OpenAPI/Swagger reference is still planned for the roadmap. Until then, this page documents the routes that currently exist in `lawallet-nwc`.

## Authentication

- `POST /api/jwt` ‚Äî Exchange a NIP-98 signed request for a JWT
- `GET /api/jwt` ‚Äî Validate a JWT
- `GET /api/jwt/protected` ‚Äî Example protected route
- `POST /api/jwt/protected` ‚Äî Example protected route

## Users and Lightning Addresses

- `GET /api/users/me` ‚Äî Load or create the current user
- `GET /api/users/[userId]/cards` ‚Äî List a user's cards
- `GET /api/users/[userId]/role` ‚Äî Read a user's role
- `PUT /api/users/[userId]/role` ‚Äî Update a user's role
- `PUT /api/users/[userId]/lightning-address` ‚Äî Assign or replace a lightning username
- `PUT /api/users/[userId]/nwc` ‚Äî Store or replace an NWC URI
- `GET /api/lightning-addresses` ‚Äî Admin list of lightning addresses
- `GET /api/lightning-addresses/counts` ‚Äî Admin summary counts
- `GET /api/lightning-addresses/relays` ‚Äî Relay information
- `GET /api/lud16/[username]` ‚Äî LUD-16 payRequest response
- `GET /api/lud16/[username]/cb` ‚Äî LUD-16 callback

## Cards

- `GET /api/cards` ‚Äî List cards
- `POST /api/cards` ‚Äî Create a card
- `GET /api/cards/counts` ‚Äî Card counts
- `GET /api/cards/[id]` ‚Äî Card detail
- `DELETE /api/cards/[id]` ‚Äî Delete a card
- `GET /api/cards/[id]/write` ‚Äî Card write payload
- `GET /api/cards/[id]/scan` ‚Äî Card scan response
- `OPTIONS /api/cards/[id]/scan` ‚Äî CORS preflight for card scan
- `GET /api/cards/[id]/scan/cb` ‚Äî Card scan callback
- `OPTIONS /api/cards/[id]/scan/cb` ‚Äî CORS preflight for scan callback
- `GET /api/cards/otc/[otc]` ‚Äî Resolve OTC value
- `POST /api/cards/otc/[otc]/activate` ‚Äî Activate OTC flow

## Card Designs

- `GET /api/card-designs/list` ‚Äî List card designs
- `GET /api/card-designs/count` ‚Äî Count card designs
- `GET /api/card-designs/get/[id]` ‚Äî Card-design detail
- `POST /api/card-designs/import` ‚Äî Import card designs

## Settings and Administration

- `GET /api/settings` ‚Äî Read settings
- `POST /api/settings` ‚Äî Update settings
- `GET /api/admin/assign` ‚Äî Read admin-assignment status
- `POST /api/admin/assign` ‚Äî Assign admin role
- `GET /api/root/assign` ‚Äî Read root-assignment status
- `POST /api/root/assign` ‚Äî Assign root/admin bootstrap role

## Remote Connections

- `GET /api/remote-connections/[externalDeviceKey]` ‚Äî Remote device bootstrap metadata
- `POST /api/remote-connections/[externalDeviceKey]/cards` ‚Äî Initialize a remote card

## Other

- `POST /api/waitlist/subscribe` ‚Äî Subscribe an email to the waitlist

## Not Yet Implemented in This Repo

The following routes and features are mentioned elsewhere in roadmap docs, but they are not part of the current API surface:

- `/api/auth/login`
- `/api/auth/refresh`
- `/api/auth/nostr`
- `/api/addresses/*`
- Redirect-management endpoints
- `/api/webhooks/*`
- `/api/payments`
- `.well-known/nostr.json`

## Auth Notes

Protected routes use either:

- `Authorization: Nostr <base64>` for direct NIP-98 authentication
- `Authorization: Bearer <jwt>` for JWT-backed access after exchanging a NIP-98 request at `POST /api/jwt`

See the [JWT Authentication guide](/docs/guides/jwt-authentication) for the current auth flow.
